# 极简菜单栏密码管理器 - 设计文档

## 概述

一个轻量级的 macOS 菜单栏密码管理应用，专注于快速查询和一键复制密码。

## 目标用户

个人用户，需要管理多个账号密码，希望快速查找和复制。

## 核心功能

### 1. 菜单栏界面

### 4. 交互流程

1. 点击菜单栏图标 或 按 `Cmd+Shift+P` → 弹出搜索窗口
2. 输入关键词 → 实时过滤列表（搜索范围：名称、用户名、备注）
3. 点击 `[📋]` → 复制密码到剪贴板，自动关闭
4. 点击 `[🔗]` → 打开网址 + 复制密码
5. 按 `Escape` 或点击窗口外部 → 关闭窗口

## 数据模型

```swift
// 保险库数据结构
struct Vault: Codable {
    let schemaVersion: Int = 1    // 数据版本，用于未来迁移
    var entries: [PasswordEntry]
}

struct PasswordEntry: Codable, Identifiable {
    let id: UUID
    var name: String           // 显示名称，如 "GitHub"
    var username: String       // 用户名/邮箱
    var password: String       // 密码（加密存储）
    var url: String?           // 网址（可选）
    var notes: String?         // 备注（可选）
    var icon: String?          // 图标 emoji（可选，默认 "🔒"）
    var isFavorite: Bool       // 收藏标记
    var createdAt: Date
    var updatedAt: Date
    // 注意：name + username 组合应唯一，用于导入时检测重复
}
```

## 安全机制

### 密钥架构

```
┌─────────────────────────────────────────────────────────────┐
│                        密钥架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户主密码 ──→ PBKDF2 派生 ──→ 主密钥 (MK)                  │
│                                    │                        │
│  随机生成 DEK ──→ AES-256-GCM 加密 vault                    │
│       │                                                     │
│       ▼                                                     │
│  MK 加密 DEK ──→ 存入 Keychain（Touch ID 保护）             │
│                                                             │
│  注意：Keychain 存储的是被 MK 加密的 DEK，不是主密码本身     │
└─────────────────────────────────────────────────────────────┘
```

**关键点：**
- **DEK (Data Encryption Key)**：随机生成的 256 位密钥，用于加密 vault.json
- **MK (Master Key)**：由用户主密码通过 PBKDF2 派生，用于加密 DEK
- **Keychain**：存储加密后的 DEK，通过 Touch ID 解锁访问

### 认证流程

```
首次启动
    │
    ▼
┌─────────────────────────────┐
│  设置主密码                  │
│  • 最少 8 个字符             │
│  • 需要确认输入              │
│  • 生成 DEK 并存入 Keychain  │
└─────────────────────────────┘
    │
    ▼
每次打开
    │
    ├──→ 有 Touch ID？ ──→ Touch ID 验证 ──→ 获取 DEK ──→ 解密
    │         │                          │
    │         └──→ 无 Touch ID ───────────┘
    │                                    │
    └────────────────────────────────────┘
                     │
                     ▼
              Touch ID 失败/不可用
                     │
                     ▼
              ┌─────────────────┐
              │  输入主密码      │
              │  • 3 次失败后延迟 │
              │  • 5 次失败锁定   │
              └─────────────────┘
```

### 安全措施

| 措施 | 说明 |
|------|------|
| 剪贴板清除 | 复制后 30 秒清除，通过 changeCount 检测是否被覆盖 |
| 渐进延迟 | 3 次失败后延迟 3 秒，4 次失败后延迟 15 秒 |
| 失败锁定 | 5 次失败后锁定 5 分钟 |
| 导入警告 | 导入时提示 JSON 包含明文密码 |
| 文件清理 | 导入成功后询问是否删除源文件 |

### 加密方案

- **算法**：AES-256-GCM
- **密钥派生**：PBKDF2-SHA256，100,000 次迭代
- **存储**：整个 vault.json 文件加密
- **位置**：`~/Library/Application Support/PasswordManager/vault.json`

## 数据导入/导出

### 导入格式（v1）

```json
{
  "version": 1,
  "entries": [
    {
      "name": "GitHub",
      "username": "myemail@example.com",
      "password": "mypassword123",
      "url": "https://github.com",
      "notes": "工作账号",
      "icon": "🔒"
    }
  ]
}
```

### 导入流程

1. 选择 JSON 文件
2. 显示警告：「导入文件包含明文密码，导入后建议删除源文件」
3. 解析并验证格式
4. 检测重复（按 name + username）
5. 导入成功后询问：「是否删除源文件？」

### 导出流程

1. Touch ID 验证
2. 导出为相同 JSON 格式（明文）
3. 显示警告：「导出文件包含明文密码，请妥善保管」

## 错误处理

### 场景与处理

| 场景 | 处理方式 |
|------|----------|
| vault.json 损坏 | 显示错误，提供从备份恢复或重新开始选项 |
| Keychain 访问失败 | 提示输入主密码重新生成 DEK |
| 存储空间不足 | 保存前检查，提示用户清理空间 |
| 导入格式错误 | 显示具体错误行，跳过错误条目继续导入 |
| 重复条目 | 显示冲突列表，选择：跳过/覆盖/保留两者 |
| 导入文件不存在 | 提示文件路径无效 |

## 数据迁移策略

未来版本如需修改数据结构：

1. 读取 `schemaVersion` 字段
2. 根据版本号执行迁移逻辑
3. 更新 `schemaVersion` 并保存
4. 保留原始数据备份（`.bak`）
